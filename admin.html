Remitente y contacto
Límites diarios

Una cuenta de Gmail gratuita (no Workspace) tiene un tope real de aproximadamente 100 correos/día para todo lo que mande este Apps Script — dejá estos límites por debajo de eso, sobre todo si el mismo día también se mandan correos reales de pedidos. Además, para proteger la cuenta, el envío de prueba tiene un cooldown fijo de 2 minutos entre cada uno, y el reenvío manual de un mismo pedido tiene un cooldown fijo de 60 segundos — esto no es configurable, se aplica siempre (incluso si alguien intentara llamar directo al webhook, sin pasar por este panel).

Roles y Permisos
Cambios sin guardar

Elegí qué puede hacer cada rol dentro del panel. Super Admin (únicamente tintinaccs@gmail.com) siempre tiene control total y no aparece como columna — no es desactivable, ni siquiera por otra cuenta con rol "superadmin" en Firestore. Usuarios, Auditoría, Correos, Configuración, Onboarding e Import/Export siguen siendo exclusivos de Super Admin (no aparecen acá como módulos editables) — el reporte de esta entrega explica por qué cada uno.

Web pública / Cliente

Las capacidades del rol Cliente (ver catálogo, agregar al carrito, comprar, ver/editar su perfil, ver sus pedidos, recibir correos) son funcionalidad base de la tienda — no un permiso de panel — así que no aparecen acá como switches. No tienen sentido como "desactivables" en general: son la razón de ser del sitio público. Para restringir a una cuenta puntual, usá Usuarios → Bloquear (ya existente), que corta todo acceso real, con Firestore Rules incluidas.

  • Ver catálogo, agregar al carrito, comprar
  • Ver y editar su propio perfil
  • Ver sus propios pedidos
  • Recibir correos de pedido / promociones (si Super Admin las activa)
